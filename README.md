#   Webオンライン対戦型ダウト大富豪
## 1. 作品概要
Webブラウザ上で動作する、オンライン対戦用の大富豪ゲームです。

※学校部活内でのグループ制作のプロジェクトの一部であり、自身が担当したゲームロジック部分を公開しています。

## 2. 担当範囲
- **担当:** ゲーム進行管理、カード提出制限ロジック
- **使用言語:** HTML(index.html),JavaScript (index.js)
- **主な実装機能:**
    - サーバとの通信(リアルタイム更新)
    - 場のカードに対して数値・枚数の大小比較

## 3. 技術的なこだわり
### 速度重視の処理
作品展示の機会が学校の文化祭で、QRコードを用意してお客様にプレイしてもらう形だったためリアルタイム性を重視しました。外部からの不正な操作をサーバで監視しリスクを減らす形ではなく、フロントエンドで役やターンを即座に判断しサーバに共有する処理を選びました。

## 4. 学んだこと
本プロジェクトを通じて、複数人での共同開発における規則とコミュニケーションの重要性を痛感しました。
制作を分担した際、グループ内での関数の動作や変数の命名規則に一貫性がなく、コード統合のための修正に時間が掛かってしまいました。結果として、当初予定していた各数字の役や細かいルール設定などの機能が搭載できませんでした。
今回の経験より、以下の2つのことを学びました。
 - 事前に関数・変数の定義をグループ内でドキュメントとして保存
 - 「単に動くコード」を書く以上に「他人が読みやすく統合しやすいコード」を書くことが重要

今後、組織のような多人数の開発にこの経験を活かしたいです。

### 生成AIの活用について
本プロジェクトの開発では、生産性とコード品質の向上のため、Gemini等の生成AIを積極的に活用しました。
AIが出力したコードを鵜呑みにせず、必ず自身の知識で動作原理を理解した上で使用しています。
